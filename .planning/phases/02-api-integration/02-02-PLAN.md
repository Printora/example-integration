   - { NextResponse } from 'next/server'
   - { verifyWebhookSignature } from '@/lib/webhook-verify'
   - { addEvent } from '@/lib/webhook-store'
   - type { PrintoraWebhookEvent } from '@/types/printora'

2. **POST handler**:
   ```typescript
   export async function POST(request: Request)
   ```

3. **Signature verification flow**:
   - Extract signature header: `request.headers.get('x-printora-signature')`
   - **CRITICAL**: Read raw body FIRST: `const rawBody = await request.text()`
   - Verify signature: `const isValid = verifyWebhookSignature(rawBody, signature)`
   - Return 401 if invalid: `NextResponse.json({ error: 'Invalid webhook signature' }, { status: 401 })`

4. **Parse and validate event**:
   - Parse JSON after verification: `const event = JSON.parse(rawBody) as PrintoraWebhookEvent`
   - Validate required fields: id, type, timestamp, data
   - Return 400 if malformed: `NextResponse.json({ error: 'Invalid webhook payload' }, { status: 400 })`

5. **Store event**:
   - Call `addEvent(event, true)` - returns boolean (false = duplicate)
   - Log successful storage with event ID and type

6. **Return response**:
   - Always return 200 OK (even for duplicates - idempotency)
   - Include duplicate flag: `{ received: true, duplicate: !added }`

Per research: "Always call req.text() before req.json() for webhook endpoints"

Per research: "The exact header name is unclear - use x-printora-signature as placeholder"
