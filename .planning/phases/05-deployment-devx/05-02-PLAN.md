# Printora Partner Integration Example

A Next.js demo/reference implementation showing how to integrate with the Printora Partner API for print-on-demand order fulfillment. This example demonstrates session creation, webhook handling, and event dashboard.

## Overview

This integration example shows the complete partner flow:
1. Create a partner session with design image and customer data
2. Redirect user to Printora's design editor
3. Receive webhook events for order lifecycle (created → paid → shipped → delivered)
4. Display webhook events in real-time dashboard

## Prerequisites

- **Node.js 20.9.0 or higher** - Required by Next.js 16
- **npm** - Package manager (comes with Node.js)
- **Printora API credentials** - Test mode key and webhook secret from [Printora Dashboard](https://staging.printora.ai/en/integrations)

## Quick Start

### 1. Clone and install

\`\`\`bash
git clone <your-repo-url>
cd example-integration
npm install
\`\`\`

### 2. Configure environment variables

\`\`\`bash
cp .env.example .env
\`\`\`

Edit `.env` and add your Printora credentials:
- `PRINTORA_API_KEY` - Your test mode API key
- `PRINTORA_API_URL` - \`https://api-staging.printora.ai\` (staging) or \`https://api.printora.ai\` (production)
- `PRINTORA_WEBHOOK_SECRET` - Your webhook secret for signature verification
- `NEXT_PUBLIC_APP_URL` - \`http://localhost:3000\` (local) or your deployed URL

### 3. Run development server

\`\`\`bash
npm run dev
\`\`\`

Visit [http://localhost:3000](http://localhost:3000) to see the landing page.

### 4. Test the integration

1. Create a session from the landing page
2. Complete the flow in Printora editor
3. View webhook events at [http://localhost:3000/dashboard](http://localhost:3000/dashboard)

## Environment Variables

| Variable | Description | Required | Default |
|----------|-------------|----------|---------|
| `PRINTORA_API_KEY` | Printora API key (test or live mode) | Yes | - |
| `PRINTORA_API_URL` | Printora API base URL | Yes | \`https://api-staging.printora.ai\` |
| `PRINTORA_WEBHOOK_SECRET` | Webhook secret for HMAC signature verification | Yes | - |
| `NEXT_PUBLIC_APP_URL` | Base URL for callbacks (local or deployed) | Yes | \`http://localhost:3000\` |

**Important:** Variables without \`NEXT_PUBLIC_\` prefix are server-side only. Never prefix sensitive values like API keys with \`NEXT_PUBLIC_\` as they become visible in the browser.

# Webhook Testing Examples

This document provides sample webhook payloads and commands for testing the Printora integration webhook handler.

## Prerequisites

- Local development server running: \`npm run dev\`
- Environment variables configured in \`.env\`
- Webhook secret from your \`.env\` file

## Signature Generation

To test webhook signature verification, you need to generate a valid HMAC-SHA256 signature.

### Using Node.js script

Add this script to \`scripts/test-webhook-signature.js\`:

\`\`\`javascript
import { createHmac } from "node:crypto";
import { readFileSync } from "node:fs";

// Read your webhook secret from .env
const envContent = readFileSync(".env", "utf-8");
const secret = envContent.match(/PRINTORA_WEBHOOK_SECRET=(.+)/)?.[1];

if (!secret) {
  console.error("PRINTORA_WEBHOOK_SECRET not found in .env");
  process.exit(1);
}

// Sample payload
const payload = JSON.stringify({
  id: "evt_test123",
  type: "order.created",
  timestamp: new Date().toISOString(),
  data: {
    orderId: "ord_test456",
    sessionId: "sess_test789",
    status: "created",
    customerEmail: "test@example.com",
    customerName: "Test Customer"
  }
});

// Generate signature
const signature = \`sha2=\${createHmac("sha256", secret)
  .update(payload, "utf8")
  .digest("hex")}\`;

console.log("Payload:", payload);
console.log("Signature:", signature);
console.log("\\nFull curl command:");
console.log(\`curl -X POST http://localhost:3000/api/webhooks \\\\\\`);
console.log(\`  -H "Content-Type: application/json" \\\\\\`);
console.log(\`  -H "x-printora-signature: \${signature}" \\\\\\`);
console.log(\`  -d '\${payload}'\`);
\`\`\`

Run with: \`node scripts/test-webhook-signature.js\`

## Testing with curl

### Valid signature test

Replace \`CALCULATED_SIGNATURE\` with signature from the generation script:

\`\`\`bash
curl -X POST http://localhost:3000/api/webhooks \\
  -H "Content-Type: application/json" \\
  -H "x-printora-signature: sha256=CALCULATED_SIGNATURE_HERE" \\
  -d '{
    "id": "evt_test123",
    "type": "order.created",
    "timestamp": "2025-02-24T10:30:00Z",
    "data": {
      "orderId": "ord_12345",
      "sessionId": "sess_67890",
      "status": "created",
      "customerEmail": "test@example.com",
      "customerName": "Test Customer"
    }
  }'
\`\`\`

**Expected response:** \`{"received":true,"duplicate":false}\` (status 200)

### Invalid signature test

\`\`\`bash
curl -X POST http://localhost:3000/api/webhooks \\
  -H "Content-Type: application/json" \\
  -H "x-printora-signature: sha256=invalid_signature" \\
  -d '{
    "id": "evt_test123",
    "type": "order.created",
    "timestamp": "2025-02-24T10:30:00Z",
    "data": {
      "orderId": "ord_12345",
      "status": "created"
    }
  }'
\`\`\`

**Expected response:** \`{"error":"Invalid webhook signature"}\` (status 401)

### Missing signature test
